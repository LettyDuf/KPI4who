Tu as raison : il existe déjà trois icônes dessinées (HTML/CSS pur) dans le cartouche de la fiche d'indicateur pour niveau · maturité · fiabilité. Ce que tu gagnes. Le panier réemploie à 100 % le langage visuel déjà établi sur la fiche : les strates horizontales pour le niveau, la jauge croissante pour la fiabilité. Zéro vocabulaire nouveau à apprendre. Le clic reste sur le nom de l'indicateur (comportement 9.B.1 préservé).

Le point qui reste à arbitrer. La chip type (IGO / ICP / ICC / ORC / DORA / FLUX / II) n'a pas d'icône dessinée sur la fiche — elle est traitée en texte seul. Trois approches possibles ci-dessous.

3. Arbitrage ouvert — comment traiter la chip TYPE ?

Fond coloré doux, code 3 lettres. C'est ce que tu as validé au précédent preview (panneau A). Pas d'icône dessinée.

Inclination. V1 reste le plus sobre et respecte ta préférence précédente. V2 mérite un coup d'œil si tu veux une vraie symétrie visuelle entre les trois chips (chacune avec son signifiant graphique), au prix d'un emoji dupliqué sur la carte (le type apparaît déjà dans le médaillon à gauche). V3 est plus marqué mais rompt l'équilibre entre les trois chips : il force un hiérarchie type ≫ niveau ≈ fiabilité.
